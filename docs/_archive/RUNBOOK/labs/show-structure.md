# Ship Club Lab — 60 min show structure

Source-of-truth: `docs/PRD.md` §4 · `docs/COMMUNITY.md` §4-5.

## Cadence

5 Labs/week. Mix of:
- **Live idea sessions** (Greenfield-oriented) — conceive + demo work
- **Live unstick sessions** (Brownfield-oriented) — clear walls + ship next version

## Pre-Lab (24 hrs prior — Radar automates)

- [ ] Community MCP reads all hot-seat submissions from the week (target: 15-50/week)
- [ ] Cluster by wall pattern
- [ ] Draft agenda: 5 hot seats + 1-3 recurring wall topics
- [ ] Ping Will's agent with draft
- [ ] Will approves in <60 sec
- [ ] Agenda published to Skool feed
- [ ] Members with matching walls receive personalized pre-brief

## Show structure (60 min)

```
0:00 — 0:03  Intro card + agenda reveal
0:03 — 0:13  Hot Seat 1
0:13 — 0:23  Hot Seat 2
0:23 — 0:33  Hot Seat 3
0:33 — 0:43  Hot Seat 4
0:43 — 0:53  Hot Seat 5
0:53 — 1:00  Wrap: library entries created + skill candidates promoted + next Lab teaser
```

## Per hot seat (10 min template)

- **0:00 — 0:01** — Seat reveal (name + wall in their own words, lower-third animation)
- **0:01 — 0:04** — Sprinter walks through what their agent tried overnight + the taste call flagged
- **0:04 — 0:08** — Will diagnoses live: reads the code / config / error together with the sprinter, points at what's off
- **0:08 — 0:09** — Sprinter directs their agent with the diagnosis; agent runs
- **0:09 — 0:10** — Fix shipped or clear next step; library entry stamp + skill candidate flag

## Camera / production

- **Screen A** (main): current hot seat's Cursor + Claude Code + terminal
- **Screen B** (PIP): Will's face cam (workshop cream backdrop, no ring light)
- **Overlay**: wall in own words + timestamp counter
- **Bug** (top right): "5 SLOTS/WK · stuck.builders"

## Post-Lab pipeline (auto)

1. Recording → Whisper transcribes
2. Community MCP tags each seat by wall category
3. Library entries auto-drafted (one per hot seat) → Will reviews + publishes
4. Skill candidates flagged for review
5. Recap post auto-drafted per `docs/SKOOL_POSTS.md` Template 3
6. YouTube edit: 60 min → 5× 3-5 min standalone episodes
7. Twitter/X: one-tweet-per-seat thread same day

## Rules

- **5 seats, no more, no fewer.** If backlog > 15 open apps, add a second weekly Lab before raising cap.
- **Not every seat ships a fix on the call.** Say the truth in the recap: 2/5 have follow-up work.
- **No fabricated wins.** If a seat didn't clear, note it as "next action logged, follow-up in async feed" — never spin.
- **Member consent per hot seat.** Recording is default; public reshare requires per-seat opt-in.
