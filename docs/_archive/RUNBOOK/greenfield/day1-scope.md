# Greenfield Day 1 — Imaginative Solution Lock (15 min call)

The PRD skeleton, background-mine draft, and imaginative solution shape are already drafted BEFORE this call (agent-to-agent handshake). Day 1 is 15-min alignment on the pre-drafted shape, not from-a-blank-page conception.

Source-of-truth: `docs/PRD.md` §3A · `docs/OFFER.md` Greenfield section · memory rule `feedback_imaginative_solution.md`.

## The Greenfield wedge — the reason this call exists

AI democratized building. Almost nobody can conceive an idea worth building.

Day 1 is the conception. The customer walks out with a shape of an idea worth building, developed with Will's taste + judgment. PRD, demo, operating environment are artifacts of that solution — the delivery vehicle, not the outcome.

**Never frame Day 1 as "validate an idea."** Frame it as "conceive an imaginative solution together."

## Pre-Day-1 handshake (T-3 to T-2 days — agents automate)

- [ ] Applicant's agent + Radar reads Q3 background + local transcripts
- [ ] Drafts proposed imaginative solution shape + PRD skeleton + Day 2-7 plan
- [ ] Radar surfaces recurring patterns from applicant's life (3+ occurrences = signal)
- [ ] Four filters pre-checked at application stage (F1 clean, F2-F4 lens applied by Will's operator agent)
- [ ] Community MCP flagged relevant demos + patterns from similar prior sprints
- [ ] Applicant reviews at `stuck.builders/scope/<id>` → approve / edit inline
- [ ] Will's operator agent reviews + surfaces to Will
- [ ] Will taps in ~5 min: Agree · Adjust · Route
- [ ] Locked PRD skeleton + imaginative solution shape sent back to applicant
- [ ] Radar + Fuse hook enabled if Ship Club member

## Call structure (15 min alignment)

### 0:00 — 0:02 · Open the locked PRD together
- Both open `stuck.builders/scope/<id>` on screen
- Confirm imaginative solution shape

### 0:02 — 0:08 · Background-mine deepens the framing
- Live conversation reveals what the pre-draft missed
- Look for the closer, better idea sitting in their own life (Radar surfaces recurring patterns; humans confirm significance)
- Update PRD inline if the conversation reveals a better direction

### 0:08 — 0:12 · Four filters cross-checked live
- F2 real problem: evidence?
- F3 valuable: earns time?
- F4 close to you: right person?
- Any red flag → we pivot to the closer, better idea from the background-mine

### 0:12 — 0:14 · Confirm the agent's Day-2 brief
- Agent's overnight instructions for building Night 2 of the demo are already drafted
- Both approve the brief
- Cover: what the demo shows Day 7, taste-call escalation path, budget cap

### 0:14 — 0:15 · Close
- Confirm 15-min daily windows on calendar
- Cover the guarantee anchor verbatim (below)

## Guarantee anchor

If you don't walk away with an imaginative solution you believe is worth building, a working AI-native demo of it, and the operating environment we built together by the Day 7 delivery call, full refund. Restate on the call at the 28-min mark.

## What Will does vs what the customer's agent does (post-Day-1)

**Will (Days 2-6):**
- 30–60 min daily planning-mode call
- Guide taste on solution shape as it evolves
- Answer the "does this direction still hold?" questions
- Pressure-test the demo against real people the customer talks to

**Customer's agent (Days 2-6):**
- Executes the demo build overnight on customer's machine
- Sets up the operating environment (prompts, docs, tools, skills)
- Flags every design/UX taste call for the customer
- Never publishes the demo without approval

## The Day 6 pivot/proceed/stop call — under the new model

- Same call, still 30-60 min
- The demo IS in front of you (agent built it overnight)
- Evidence: what real people said about the shape
- Verdict: proceed to Brownfield / pivot with the demo as artifact / stop
- Verdict is a taste call Will owns — but supports the customer's judgment
