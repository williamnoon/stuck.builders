# Greenfield Day 7 — Delivery (30 min call)

Source-of-truth: `docs/PRD.md` §3A · `docs/OFFER.md` Greenfield · memory `feedback_imaginative_solution.md`.

## The deliverable — reminder

Not "we validated your idea." Not "here's a PRD."

**Deliverable: the imaginative solution** — an idea worth building, shaped with Will's taste. PRD + working demo + operating environment are the ARTIFACTS of the solution.

## Prerequisites

- [ ] Agent has finished the AI-native demo (link + local run instructions)
- [ ] PRD (1–2 pg) drafted
- [ ] Operating environment configured (prompts, docs, tools, skills tailored to their idea + stack)
- [ ] Next-3-steps handoff drafted
- [ ] Radar receipts of the imaginative-solution-shaping process

## Call structure (30 min)

### 0:00 — 0:03 · Frame the deliverable
- "You applied with a question. You leave with an imaginative solution and the working proof."
- Restate the operator identity: "We're builders building with what's next-gen. We show you how to prove — to yourself, your team, your market — that you can engineer and orchestrate real-world agents that ship."

### 0:03 — 0:15 · Demo the AI-native MVP
- Sprinter opens the demo (running locally or on a preview URL)
- Walks through: what it does, who it's for, why it works
- Verify: the demo shows the imaginative solution, not just "a thing that runs"

### 0:15 — 0:22 · PRD walkthrough
- Open the 1-2 pg PRD
- Confirm: dream outcome, scope, next-version definition
- Note what's IN scope and what's out (guardrails)

### 0:22 — 0:26 · Next-3-steps handoff
- If proceeding: Brownfield Build is the natural next door — walk in with PRD + demo already in hand
- If pivoting: the demo IS the pivot artifact
- If stopping: you leave with the year back + the learnings + the operating environment for the next thing

### 0:26 — 0:28 · Operating environment handoff
- Prompt packs configured for their stack
- Skills library seeded with relevant items
- Fuse hook enabled if Ship Club
- Radar wired to their next project

### 0:28 — 0:30 · Skool pitch (verbatim from COMMUNITY.md §6)

## Guarantee anchor — restate at the 15-min mark

"If you don't walk away with an imaginative solution you believe is worth building, a working AI-native demo of it, and the operating environment we built together — full $199 refund. A 'don't build this' verdict — with the demo as a pivot artifact — still counts as delivery, but if you don't get the deliverables at all, you don't pay."

## What Will does vs the sprinter

**Will:**
- Verifies the demo demonstrates the imaginative solution
- Ships the PRD in final form on the call
- Runs the Skool pitch

**Sprinter:**
- Demos + articulates their own solution
- Decides on proceed/pivot/stop
- Takes ownership of the artifacts (they're theirs forever)

## Post-call artifacts

- [ ] PRD + demo link + operating environment saved to sprinter's `~/greenfield/YYYY-MM-DD-<slug>/`
- [ ] Ship Log entry drafted
- [ ] 30-day check-in queued
- [ ] Community skill contributions committed if opt-in
- [ ] If proceeding to Brownfield: apply link + acceptance pathway pre-approved based on their Greenfield receipts

## Never

- Never guarantee the idea will make money
- Never spin a "don't build this" as sales language
- Never fabricate a demo — the demo shows what the imaginative solution IS
