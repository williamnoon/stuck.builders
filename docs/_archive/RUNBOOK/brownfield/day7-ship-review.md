# Brownfield Day 7 — Ship review (30 min call)

Source-of-truth: `docs/PRD.md` §3B · `docs/OFFER.md` Brownfield guarantee.

## Prerequisites (agents automate these)

- [ ] Agent has deployed the target next version to a real URL (staging or prod per sprinter's approval)
- [ ] Next-version roadmap doc drafted
- [ ] Session recordings archived to sprinter's Radar
- [ ] Skill candidates flagged from the sprint's 7 sessions
- [ ] Library entries auto-drafted for each wall cleared

## Call structure (30 min)

### 0:00 — 0:05 · Demo the deployed app
- Sprinter opens the live URL
- Walks through: what changed, what shipped, what works
- Verify: `deployed = target from Day 1 Version Map` — non-negotiable

### 0:05 — 0:15 · Review the Version Map delivery
- Open Day 1 map next to today's state
- Check each milestone
- Note any that shifted (with reason)
- The guarantee anchor: "If your next version isn't live on a real URL by the Day 7 ship review — deployed, working, visitable — full $399 refund."
- If the target isn't shipped: refund conversation, not spin

### 0:15 — 0:20 · Next-version roadmap handoff
- What's the next wall you'll hit?
- Roadmap doc reviewed: "when the next wall shows up, here's the map"
- Session recordings + operating environment + Radar receipts all keep

### 0:20 — 0:25 · Skill + library review (Radar surfaces)
- Radar shows: 3-5 skill candidates from this sprint
- Sprinter approves for their personal skills library
- If Ship Club: opt-in publish to community with attribution

### 0:25 — 0:30 · The Day-7 Skool pitch (verbatim from COMMUNITY.md §6)
- One shot, at the end, no repeats
- Frames Ship Club: $69/mo or $500/yr with 7-day free trial
- 30/60/90 check-ins default; no upsell inside them

## What Will does vs the sprinter

**Will:**
- Verifies shipped state matches Day 1 target
- Runs the Skool pitch
- Approves community-publishable skill candidates
- Makes the refund call if target not hit

**Sprinter:**
- Demos their live URL
- Approves their personal library additions
- Decides on Ship Club (optionally, over the coming week — no pressure)

## Post-call artifacts

- [ ] Deployed URL logged to sprinter's receipts
- [ ] Next-version roadmap saved to their Radar
- [ ] Skill candidates promoted to personal library
- [ ] Community skill contributions committed (if opt-in)
- [ ] Post-sprint 30/60/90 check-in queued (auto, calendar drops)
- [ ] Ship Log entry drafted (celebration + receipt, sprinter approves before Skool feed)

## Failure mode — what if the target didn't ship?

- Refund fires per guarantee
- Not "spin as learning opportunity"
- Session recordings still theirs
- Post-mortem in the next 48 hrs (optional, both agree) → what should we have scoped differently
- This informs future Day 1 templates — feed the lesson back into `brownfield/day1-scope.md`
